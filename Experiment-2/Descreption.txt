STUDENT REGISTRATION FORM TEST CASES & VALIDATIONS
1. Field: Name
Requirement:
This field is required (cannot be left empty).
Must contain only uppercase/lowercase alphabets and spaces.
Pattern used: [A-Za-z\s]+

Valid Inputs :
"Deepjoy Paul"
"Silvester Stallone"

Invalid Inputs :
"deepjoy123" → Contains numbers (violates pattern)
"deepjoy_paul" → Contains an underscore (not allowed)
"DEEPJOY@paul" → Contains a special character

2. Field: Email Requirement:
This field is required.
Must follow standard email format like example@domain.com or example@domain.co.in

Valid Inputs:
"deepjoypaul442@gmail.com"
"deepjoy442@gmail.co.in"

Invalid Inputs:
"deepjoy.paul" → Missing @ and domain
"deepjoy@.com" → Domain name is missing
"deepjoy@com" → Missing . in domain
"deepjoy@domain,com" → Comma instead of dot

3. Field: Age
Requirement:
This field is required
Must be numeric
Must be between 5 and 130

Valid Inputs:
5 → Minimum valid
18, 65 → Typical valid ages
130 → Maximum valid

Invalid Inputs:
"" (empty) → Triggers "required" validation
4 → Below minimum range
131 → Above maximum range
"abc" → Non-numeric input

4. Form Submission
If all fields are valid:
The form will submit successfully.
If any field is invalid:
The form will not submit, and the browser displays HTML5 validation messages for correction.
